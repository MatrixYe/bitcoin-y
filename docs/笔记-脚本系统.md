Bitcoin Script 本质上是一个基于栈的、非图灵完备的小型解释器。脚本系统不是“任意脚本语言”，而是一个专门服务于交易花费验证的栈机。

1. 一笔交易的输入要花费某个旧输出
2. 旧输出里有 script_pubkey，它定义“怎样才能花这笔钱”
3. 新交易的输入里有 script_sig，它提供“满足条件所需的数据”
4. 验证时，节点先执行 script_sig，再执行被引用输出的 script_pubkey
5. 如果最终栈顶为真，脚本验证通过；否则失败
