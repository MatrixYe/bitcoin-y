# 比特币交易序列化与哈希计算分析
比特币交易分 ** 传统（非隔离见证）与隔离见证（SegWit）** 两种序列化格式，txid 计算只使用传统格式，wtxid 使用带见证的完整格式。

## 交易序列化流程

比特币交易的序列化主要在 `CTransaction` 类中实现，核心文件是 `primitives/transaction.h` 和 `primitives/transaction.cpp`。

### 1. 序列化方法

交易的序列化通过 `Serialize` 方法实现：

```cpp
template<typename Stream>
void CTransaction::Serialize(Stream& s) const
{
    SerializeTransaction(s, *this);
}
```

### 2. 序列化实现

`SerializeTransaction` 函数负责具体的序列化逻辑：

```cpp
template<typename Stream>
void SerializeTransaction(Stream& s, const CTransaction& tx)
{
    const int32_t nVersion = tx.nVersion;
    const bool fOverwintered = tx.HasWitness();
    ::Serialize(s, nVersion);
    if (fOverwintered) {
        ::Serialize(s, (int32_t)1); // nVersionGroupId
    }
    ::Serialize(s, tx.vin);
    ::Serialize(s, tx.vout);
    if (fOverwintered) {
        ::Serialize(s, tx.witness);
    }
    ::Serialize(s, tx.nLockTime);
}
```

### 3. 序列化字段顺序

交易序列化的字段顺序为：
1. **版本号** (`nVersion`) - 4字节
2. **版本组ID** (`nVersionGroupId`) - 仅当有见证数据时，4字节
3. **输入列表** (`vin`) - 可变长度
4. **输出列表** (`vout`) - 可变长度
5. **见证数据** (`witness`) - 仅当有见证数据时，可变长度
6. **锁定时间** (`nLockTime`) - 4字节

### 2. 序列化哈希实现

`SerializeHash` 函数是通用的序列化哈希计算函数：

```cpp
template<typename T>
uint256 SerializeHash(const T& obj, int nType=SER_GETHASH, int nVersion=PROTOCOL_VERSION)
{
    CHashWriter ss(nType, nVersion);
    ss << obj;
    return ss.GetHash();
}
```

### 3. 哈希写入器

`CHashWriter` 是一个特殊的序列化器，它在序列化过程中计算哈希值：

```cpp
class CHashWriter
{
private:
    CSHA256 ctx;

public:
    int nType;
    int nVersion;

    CHashWriter(int nTypeIn, int nVersionIn) : nType(nTypeIn), nVersion(nVersionIn) { }

    template<typename T>
    CHashWriter& operator<<(const T& obj)
    {
        ::Serialize(*this, obj);
        return *this;
    }

    void write(const char* pch, size_t nSize)
    {
        ctx.Write((const unsigned char*)pch, nSize);
    }

    uint256 GetHash()
    {
        uint256 result;
        ctx.Finalize((unsigned char*)&result);
        return result;
    }
};
```

## 见证交易的特殊处理

### 1. 见证交易的哈希计算

对于包含见证数据的交易，有两种哈希：
- **常规哈希** (`GetHash()`) - 包含见证数据
- **见证哈希** (`GetWitnessHash()`) - 不包含见证数据，用于共识验证

```cpp
uint256 CTransaction::GetWitnessHash() const
{
    return SerializeHash(*this, SER_GETHASH | SER_WITNESS, 0);
}
```

### 2. 序列化标志

- `SER_GETHASH` - 用于计算哈希的序列化标志
- `SER_WITNESS` - 表示包含见证数据的标志

## 输入输出示例

#### 输入输出示例

**输入**：一个简单的交易
```
交易版本: 2
输入数量: 1
输入1:
  交易ID: 0x1234567890abcdef...
  输出索引: 0
  脚本: "0x146666666666666666666666666666666666666666"
  序列: 0xffffffff
输出数量: 1
输出1:
  金额: 5000000000 (50 BTC)
  脚本: "0x76a914000000000000000000000000000000000000000088ac"
锁定时间: 0
```

**序列化过程**：
1. 写入版本号: `0x02000000`
2. 写入输入数量: `0x01`
3. 写入输入1:
    - 交易ID: `0x1234567890abcdef...`
    - 输出索引: `0x00000000`
    - 脚本长度: `0x19`
    - 脚本: `0x146666666666666666666666666666666666666666`
    - 序列: `0xffffffff`
4. 写入输出数量: `0x01`
5. 写入输出1:
    - 金额: `0x00f2052a01000000`
    - 脚本长度: `0x19`
    - 脚本: `0x76a914000000000000000000000000000000000000000088ac`
6. 写入锁定时间: `0x00000000`

**哈希计算**：
1. 使用 `CHashWriter` 序列化交易
2. 对序列化结果计算 SHA-256 哈希
3. 再对结果计算一次 SHA-256 哈希（双哈希）
4. 得到交易哈希值
